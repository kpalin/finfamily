FinFamily

Swing-based java-application for managing genealogical data.

The genealogical data is stored in a PostgreSQL database

The main class for the swing application is {@link fi.kaila.suku.swing.Suku Swing entry}.

The main class for the webstart application is {@link fi.kaila.suku.webstart.Suku Webstart entry}.

Architecture

The application architecture consists of following main components:

  • User interface consisting of swing displays
  • A Kontroller that connects the User interface to the server application
  • Server application that manages the PostrgreSQL database.

FinFamily requires Java 6SE and the PostgreSQL database. At least Windows and Linux systems support these. Possibly the application can be used on a Mac computer.

PostgreSQL is a real database. It is the most advanced open source relational database available. Both Java and PostgreSQL are free and makes the application affordable. Both have easy to use installation procedures but the installation of these may require the support of an IT professional.

Flavours

  • Local use. Both userinterface and database are installed on same workstation. (default)
  • Client-Server use. Application is installed on the local workstation and PostgreSQL database is installed on a database located in teh same network.
  • WebStart use. Application and server are installed as webstart application on the webserver on e.g. tomcat application server. Application is started from browser and uses the JNLP protocol between user and server.
